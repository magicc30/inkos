# Studio 剧情多线推演设计

## 目标

把 #342 已有的结构化推演结果直接呈现在长篇书籍 Chat 中，让作者比较候选未来并保存一个候选计划；不新增工作台，不进入 Play，不修改正史。

## 产品边界

- 推演卡只消费 `create_narrative_forecast` 与 `get_narrative_forecast` 返回的结构化 `details`。
- 候选分支始终标记为非正史。
- “采用此分支”只调用 `select_narrative_branch`，生成 `selected-branch-plan.md`。
- 采用动作不改章节、大纲、`current_focus.md`、`current_state.md`、伏笔或结构化状态。
- 过期推演禁用采用动作，作者需先重新推演。

## 交互

推演卡顶部显示分歧点、基准章节、推演跨度、分支数和状态。桌面端使用横向分支轨道，移动端纵向排列。

每个分支的首屏只显示：

- 标题与意图匹配度
- 分支前提
- 未来章节节拍
- 风险数量与风险类型

人物决定、预计变化、不确定性和匹配理由放在可展开详情中，避免三张卡同时变成长报告。

采用按钮从卡片发出明确的按钮动作，请求现有 Agent 调用 `select_narrative_branch`。选择结果继续作为工具消息留在会话中，保持 Chat 的审计链。

## 视觉方向

- Domain：作者在写下一章前使用的“分叉编辑桌”。
- Color world：沿用 InkOS 的墨黑、纸面灰和金色强调，不引入 Play 的游戏色。
- Signature：顶部一条分叉轨迹连接候选分支，强调“同一起点、互不混写”。
- Reject：独立预测后台、Play 选项按钮、通用数据仪表盘卡片。

## 错误与过期

- 缺失或无效的 `details` 回退到现有工具结果文本，不让聊天渲染崩溃。
- `stale` 卡展示醒目警告并禁用采用按钮。
- 采用失败由现有工具错误卡显示，不把失败伪装成已选择。

## 验收

- 创建与读取推演能渲染同一套比较卡。
- 三分支在桌面可横向比较，移动端可完整阅读。
- 过期结果不能采用。
- 点击采用后出现 `select_narrative_branch` 工具结果，且正史文件不变。
- 会话刷新后结构化卡仍可恢复。
